### Cohérence Migrations ↔ JPA ↔ DTOs

**Cohérent** : Les 8 migrations produisent exactement le schéma reflété par les entités JPA. Les FK SET NULL correspondent aux `@ManyToOne(optional = true)`. ## 4. FLUX CRITIQUES

### A. Inscription et vérification

OAuth consent → CustomOAuth2UserService → AuthService.processOAuth2Login (create User + UserProfile) → JwtTokenProvider.generateToken → OAuth2LoginSuccessHandler (cookie HttpOnly/Secure/Lax 7j) → Redirect frontend → useAuthInit → GET /me → loginFromUser → TermsGate → acceptTerms → requestVerification (Redis code 15min TTL + email) → confirmVerification (5 tentatives max, hash SHA-256, nouveau JWT).

### B. Upload document

POST multipart → @RateLimit 5/jour → Content-Type PDF + magic bytes %PDF- + 10MB max → PdfCompressionService (Ghostscript /ebook 60s) → MinioService.upload (UUID key) → Document save + tags sanitisés (HTML-escape, lowercase, trim) → MeilisearchService.indexDocument (async, retry 3x) → addXp(10).

### C. Téléchargement

GET /documents/{id}/file (VERIFIED) → Redis INCR dl-buffer:{id} → addXp(1) auteur (anti-farming si même user) → MinioService.download → Content-Disposition: inline. @Scheduled(5min) flushDownloadCounts : Redis SCAN → incrementDownloadCount batch.

### D. Suppression compte

DELETE /me → reportRepository.findByUserId → detach reports (user=null, saveAll) → documentRepository.anonymizeByUserId → userRepository.delete (cascade ALL : profile, ratings, favorites, badges; no cascade : donations, delegates → SET NULL DB) → clearJwtCookie. Log info.

### E. Don Ko-fi

POST /webhooks/kofi → verify token → type Donation/Subscription → match user (email hash ou username) → adFreeUntil = max(now, existing) + 30j → Donation save → 200 OK (toujours).
